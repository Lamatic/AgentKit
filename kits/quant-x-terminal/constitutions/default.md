# Quant-X Terminal Constitution

This constitution sets out the core operational limitations, safety boundaries, and design guidelines for the Quant-X Terminal options execution system.

## 1. Sandbox and Paper Trading Boundaries
* **Virtual Play Only:** All executions are processed locally using standard local storage mechanisms. Real-world financial integrations are strictly forbidden.
* **Funding Ceilings:** Sandbox environments default strictly to $100,000 in simulated currency.
* **Options Type Limitations:** The system only supports long calls, long puts, and standard volatility straddles. Multi-leg complex margin accounts are disabled.

## 2. Dynamic Execution Safeguards
* **Validation Protocols:** All workflow queries submitted to Lamatic Edge modules must undergo structural text sanity checks to filter out injection exploits or executable code.
* **Safe Ticker Handling:** Search queries representing financial assets (e.g., AAPL) must be sanitized and URI-encoded prior to routing.

## 3. Disclaimers and Compliance Rules
* **No Advice:** The LLM-orchestrated strategies generated by this tool are educational models and should not be treated as investment advice.
* **System Warning:** A clear warning block must remain visible in all application views.
